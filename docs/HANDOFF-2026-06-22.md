## 3. Política de trabalho a partir de agora (a regra principal deste handoff)

**A `main` é PRODUÇÃO.** Fluxo obrigatório:

1. Criar uma **branch** para a melhoria/feature (`git checkout -b feat/...`).
2. Implementar + rodar `lint`/`tsc`/`test` localmente.
3. `git push` da branch → a Vercel cria um **Preview Deployment** com URL
   própria. O app da Vercel no GitHub **comenta o link no PR**.
   - O agente consegue **recuperar esse link via `gh`** (GitHub deployments/
     statuses) e trazer na conversa, **sem precisar da Vercel CLI**. Plano B:
     o usuário cola o link do PR.
4. Usuário valida no preview.
5. Aprovado → merge na `main` → **deploy de produção automático**.

Exceção: docs/handoff podem ir direto à `main` quando o usuário pedir.
